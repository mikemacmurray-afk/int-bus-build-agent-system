## Extraction Process

### Step 1: Identify Brick Types

Look for these distinct patterns in the raw input:

#### **INSIGHT** üí°
A realization, discovery, or "aha moment"

**Patterns:**
- "I realized that..."
- "What I'm noticing is..."
- "The breakthrough was..."
- "I finally understood..."
- "It clicked when..."

**Example:**
*"I realized my clients don't fail because they lack tactics - they fail because they lack a decision-making framework. Tactics are infinite, but good judgment is scarce."*

#### **METAPHOR** üé®
A comparison that illuminates a concept

**Patterns:**
- "It's like..."
- "Think of it as..."
- "X is the Y of Z"
- "Imagine..."
- Visual comparisons

**Example:**
*"Your personal brand is an appreciating asset. Most people treat it like a depreciating expense - something that loses value over time unless constantly refreshed. But real brands compound. Every piece of work adds to the foundation."*

#### **DISTINCTION** ‚ö°

- "X ‚â† Y"

#### **REFRAIN** üîÅ

**Patterns:**

#### **QUESTION** ‚ùì

**Example:**

### Step 2: Extract Components

For each brick, extract:

**Required:**
- **Content**: The actual insight/metaphor/distinction (verbatim when possible)
- **Type**: insight, metaphor, distinction, refrain, framework, question
- **Source**: Where it came from (voice note, book, conversation, etc.)

**Context:**
- **Origin Story**: What triggered this thought
- **Theme**: Broad category (systems-thinking, brand-philosophy, etc.)
- **Connected Ideas**: Related bricks or concepts
- **Applications**: Where this could be used

**Metadata:**
- **Tags**: Keywords for searchability
- **Quality Score**: 1-10 based on novelty and usefulness
- **Recency**: When captured
- **Usage Count**: How many times referenced

### Step 3: Assess Quality

Rate each brick on these dimensions:

**Novelty (1-10):**
- 1-3: Common knowledge, widely known
- 4-7: Somewhat unique angle
- 8-10: Original insight, rarely expressed this way

**Utility (1-10):**
- 1-3: Interesting but not actionable
- 4-7: Useful in specific contexts
- 8-10: Immediately useful, broadly applicable

**Clarity (1-10):**
- 1-3: Vague or confusing
- 4-7: Clear with context
- 8-10: Crystal clear, standalone

**Memorability (1-10):**
- 1-3: Forgettable phrasing
- 4-7: Somewhat memorable
- 8-10: Sticky, quotable, repeatable

**Overall Quality = Average of four scores**

### Step 4: Identify Themes

Group bricks into thematic clusters:

**Common Themes:**
- systems-thinking
- brand-philosophy
- creative-process
- productivity-beliefs
- client-psychology
- personal-evolution

### Step 5: Create Connections

Link related bricks:

**Connection Types:**
- **Builds on**: Brick B extends Brick A
- **Contrasts with**: Brick B provides opposite view to Brick A
- **Example of**: Brick B illustrates concept in Brick A
- **Combines with**: Brick A + Brick B create new insight
- **Supports**: Brick B provides evidence for Brick A

## Processing Logic

### When Processing Voice Notes:

```
1. Transcribe audio (in production environment)
2. Identify natural speech markers:
   - "You know what I realized..."
   - "So here's the thing..."
   - "What's interesting is..."
3. Extract distinct ideas (not full paragraphs)
4. Preserve conversational tone in bricks
5. Remove filler words but keep character
6. Flag repeated phrases as potential refrains
7. Classify each extracted thought
8. Score quality
9. Suggest themes
10. Create narrative bricks
```

### When Processing Text:

```
1. Parse structure (paragraphs, bullets, etc.)
2. Identify complete thoughts vs fragments
3. Look for brick type indicators
4. Extract verbatim when possible
5. Add context from surrounding text
6. Classify and tag
7. Score quality
8. Connect to existing bricks
9. Store in library
```

### When Processing Book Notes:

```
1. Separate user's thoughts from author's
2. Look for synthesis ("This connects to...")
3. Identify personal applications
4. Extract user's reframes/interpretations
5. Tag with book source
6. Link to relevant existing bricks
7. Note if user is building on author's idea
```
